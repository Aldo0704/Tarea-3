Metodología BEM

BEM (Block, Element, Modifier o Bloque, Elemento, Modificador) es una metodología ágil de desarrollo basada en componentes. Fue creada por la empresa Yandex para desarrollar sitios en poco tiempo y con un soporte de largo plazo. El objetivo final de BEM es dividir la interfaz de usuario en bloques independientes para crear componentes escalables y reutilizables. Propone un estilo descriptivo para nombrar cada una de las clases a utilizar, permitiendo así crear un estructura de código consistente.

Para lograr su objetivo, la metodología se centra en la reutilización de código. Además, busca otorgar un mayor entendimiento hacia qué nos referimos cada vez que construimos un nuevo proyecto. Como resultado, crea una documentación explícita en cada nueva declaración. BEM tiene como horizonte modularizar lo máximo posible cada uno de los bloques de código dispuesto. Se centra en tres parámetros o variables posibles: Bloques (div, section, article, ul, ol, etc.), elementos (a, button, li, span, etc.) y modificadores. Estos últimos se definen de acuerdo a la posterior utilización que haga el desarrollador a cargo.

Bloques, Elementos y Modificadores (BEM)

Bloques

Es un contenedor donde se encontrarán los diferentes elementos. Por ejemplo, un encabezado (header), una barra lateral (sidebar/aside) un área de contenido principal (main) y un pie de página (footer), se consideraría cada uno como un bloque.

El bloque de elementos corresponde a la raíz de la clase y deberá ir siempre primero. Cuando ya está definido es posible nombrar los elementos que compondrán la estructura final y su contenido.

Elementos

El elemento es una de las variadas piezas que compondrán la estructura de un bloque. El bloque es el todo y los elementos son las piezas de este bloque.

Modificadores

El ejercicio de nombrar las clases sirve para que los elementos puedan repetirse. Si los elementos son los mismos, no será necesario escribir nuevas clases en otras áreas del sitio.

Ventajas de usar la Metodología BEM

La metodología BEM se encuentra en constante evolución, pero ha demostrado su eficiencia al establecer una buena jerarquía en el desarrollo de plataformas.

En el tiempo que llevamos usándola hemos percibido las siguientes ventajas:

  1. Añade especificidad: Usa un selector único para cada regla, lo que permite reducirla y hacer menos repeticiones.
  2. Aumenta la independencia: Los bloques se pueden mover a cualquier parte del documento, sin afectar los estilos.
  3. Mejora la herencia múltiple: Se puede cambiar cualquiera de las tres partidas sin afectar a las demás.
  4. Permite la reutilización: Es posible reciclar ciertas áreas de código desde un proyecto hacia otro, esto debido a la no existencia de dependencias mayores en cuanto a la implementación de cada uno de los bloques estructurados.
  5. Entrega simplicidad: Permite un fácil entendimiento una vez conocido el proceso lógico sobre el cual se basa. A su vez, las convenciones a la hora de nombrar las clases permiten tener un control absoluto al saber a qué, quién y hacia dónde hacemos referencia dentro de una estructura.

CamelCase

El camelcase es un estilo de escritura utilizado en la programación para nombrar variables y funciones. Su nombre proviene de la apariencia que tiene una palabra en este formato, similar a las jorobas de un camello. Este estilo es especialmente popular en lenguajes como Java, JavaScript, C#, y muchos otros.

El camelcase no solo se limita a la programación, sino que también se ha extendido a otras áreas, como la escritura de nombres de dominio en la web, la definición de identificadores en XML o la creación de hashtags en redes sociales. Su versatilidad y simplicidad lo convierten en una opción atractiva para mantener la coherencia y claridad en la comunicación, tanto en el ámbito técnico como en el cotidiano. Su popularidad y amplio uso en diversas plataformas lo hacen una habilidad valiosa para cualquier aspirante a desarrollador o profesional del mundo digital.

En el lenguaje de programación Java, el camelCase se usa ampliamente para nombrar variables, métodos y clases, siguiendo dos convenciones principales. El Upper Camel Case, también llamado PascalCase, se utiliza para nombrar clases y tipos. En este estilo, cada palabra del identificador comienza con mayúscula, incluida la primera. Esto permite diferenciar visualmente las clases y estructuras principales dentro del código. Por otro lado, el Lower Camel Case se emplea para nombrar variables y métodos. En este caso, la primera palabra inicia con minúscula y las siguientes con mayúscula. Este formato es común no solo en Java, sino también en otros lenguajes como JavaScript y C#, ya que mejora la legibilidad y mantiene una convención clara dentro del código.

Situaciones donde se usa camelCase

  1. Se utiliza para que los identificadores se vuelven más legibles y fáciles de entender para cualquier persona que esté revisando el código. Esto hace que sea más sencillo identificar la función y el propósito de cada variable, método o clase, lo que a su vez simplifica el proceso de mantenimiento del código a largo plazo. Los desarrolladores pueden entender rápidamente qué hace cada parte del código, lo que reduce el tiempo necesario para realizar cambios o correcciones y disminuye la probabilidad de introducir errores.
  2. El camelCase ayuda a mantener una nomenclatura coherente en los proyectos, lo que facilita la colaboración entre desarrolladores. Al seguir las mismas reglas, se mejora la comunicación y comprensión del código, especialmente en proyectos grandes. Esta consistencia evita confusiones y errores, haciendo el desarrollo más eficiente y ordenado.
  3. El camelCase evita confusiones entre variables con nombres similares al diferenciarse también por el uso de mayúsculas. Esto ayuda a mantener la integridad del código y previene errores que podrían causar fallos o comportamientos inesperados en el programa.

¿Cuándo usar BEM?

Usa BEM cuando estés trabajando con HTML y CSS, especialmente si:

  • Estás creando sitios web con muchos estilos.
  • Quieres que tus clases CSS estén bien organizadas.
  • Estás trabajando en equipo y todos deben entender las clases.
  • Quieres evitar conflictos o repeticiones de nombres de clases.

¿Cuándo usar camelCase?

Usa camelCase cuando estés programando con Java, JavaScript, Python, C#, etc., especialmente para:

  • Nombrar variables, funciones, métodos o objetos.
  • Seguir buenas prácticas de escritura de código.
  • Hacer que los nombres sean fáciles de leer.
